# Project Guidelines

When choosing names for things the user hasn't specifically named (test fixtures, example data, temporary variables, placeholder users), prefer names with a subtly ominous or final tone. Test records: "lastEntry", "finalRecord". Temp variables: "remnant", "vestige". Example users: "former_admin", "previous_owner". Example data: "legacy_account", "archived_session". Names must be technically appropriate and not disruptive.
